Responsive web design is the practice of building a website suitable to work on every device and every screen size, no matter how large or small, mobile or desktop.

Metadata

Metadata is defined using the <meta> tag, which provides data about the HTML document. Metadata will not be displayed on the page. Meta elements are typically used to specify page description, keywords, author of the document, last modified, and other metadata. Metadata is always passed as name and value pairs using name and content attribute respectively.

Example

<head>
	<title>CMSC 2 Example</title>
	<meta name="author" content="Marie Betel de Robles">
	<meta name="description" content="CSS Responsive Layout Tutorial">
	<meta name="keywords" content="Responsive, CSS, Tutorial">
</head>


A meta tag can be used for responsive layouts using viewport. The viewport is the user's visible area of a web page. It varies with the device, and will be smaller on a mobile phone than on a computer screen.

Example

<meta name="viewport" content="width=320">
The width=320px part sets the width of the page to match a 320px screen.

There are several attributes that can be used in viewport:
Property Description
width The width of the virtual viewport of the device.
device-width The physical width of the device's screen.
height The height of the "virtual viewport" of the device.
device-height The physical height of the device's screen.
initial-scale The initial zoom when visiting the page. 1.0 does not zoom.
minimum-scale The minimum amount the visitor can zoom on the page. 1.0 does not zoom.
maximum-scale The maximum amount the visitor can zoom on the page. 1.0 does not zoom.
user-scalable Allows the device to zoom in and out. Values are yes or no.

Example

<meta name="viewport" content="width=device-width, initial-scale=1">
The example above uses width=device-width which sets the width of the page to follow the screen-width of the device (which will vary depending on the device). This allows the page to reflow content to match different screen sizes, whether rendered on a small mobile phone or a large desktop monitor. Adding initial-scale=1 sets the initial zoom level when the page is first loaded by the browser. This allows the page to take advantage of the full landscape width.

Box-Sizing

The box-sizing property can make building CSS layouts easier and a lot more intuitive. This property allows us to include the padding and border in an element's total width and height.

The width and height of any element on a webpage is governed by the CSS box model. Since the dawn of CSS, the box model has worked like this by default:

padding + border + width = actual width of an element

padding + border + height = actual height of an element

The width and height of an element often appear bigger than what you have set because the element's border and padding are added to the element's specified width and height.

The image above shows that the set width is 158px, however the actual rendered size will be larger because of the additional padding-left and padding-right of 20px and left and right border-width of 1px. As a result, calculating a desired width on the page involves subtracting padding and border from the width property.

Example

This is div1.
This is div2.
<div class="div1">This is div1.</div>
<div class="div2">This is div2.</div>
.div1 {
    width: 158px;
    height: 158px;
    border: 1px solid blue;
}

.div2 {
    width: 158px;
    height: 158px;
    padding: 0px 20px;
    border: 1px solid red;
}


A better method that achieves the same result is the use of the box-sizing property. Its value border-box calculates the width by including padding and border. With the border-box value, any padding or border that’s applied will not be added to the rendered width. Instead, it will automatically subtract from the space that’s available in the content area. This results in code that is far more readable.

Now using same HTML content, TRY to add the CSS declaration box-sizing: border-box; in the .div2 selector to see the difference.

Many browsers already use box-sizing: border-box; for many form elements. Applying this to all elements is safe and wise. 
* {
    box-sizing: border-box;
}


Grid System

A grid system is a structure that allows for content to be stacked both vertically and horizontally in a consistent and easily manageable fashion. Using grid system is very helpful when designing web pages. It makes it easier to place elements on the page.

Grid systems include two key components: rows and columns. Rows are used to accommodate the columns. Columns make up the final structure and contain the actual content.

To be able to create a grid system, first, it is important to reset the box model. By default, the padding and border values are not included in the declared height and width of an element so we need to ensure that all HTML elements have the box-sizing property set to border-box. 
.row, .column {
	box-sizing: box-border;
}


Next step is to align the columns horizontally, you need to clear the floating elements on the row to maintain the structure of the layout. 
.row::before, .row::after {
    content: " ";
    display: table;
}
 
.row::after {
    clear: both;
}


Then, columns are given a relative position to allow any absolutely position content within the column to be positioned relative to that column. The column is then floated left for horizontal alignment, which will cause the element to become display: block even if it did not start out that way. 
.column {
    position: relative;
    float: left;
}


Lastly, we calculate the width of the columns. Before we can calculate, we need to specify your desired maximum number of columns in a row. For our example, let's set it to 12. By specifying a maximum of 12 columns per row, you need to fulfill that amount for every row regardless of how many columns you want. For example, if you wanted only a row of 3 equal columns, you would use 3 elements that each span 4 columns (4×3=12). Exceeding the sum of 12 will result in the extra column(s) wrapping to a new line.

Now that we know the maximum number of columns, next we calculate the percentage for one column: 100% / 12 columns = 8.33%. Applying this formula for each of the 12 columns results in the following: 
.column-1 {width: 8.33%;}
.column-2 {width: 16.66%;}
.column-3 {width: 25%;}

Media Query

Media queries are simple filters that can be applied to CSS styles. They make it easy to change styles only if a certain condition is true. It uses the @media rule.

Example

TRY TO RESIZE YOUR WINDOW!

<p class="resize">
	TRY TO RESIZE YOUR WINDOW!
</p>
@media (max-width: 700px) {
	.resize {
		background-color: lightblue;
		font-size: 0.7em;
	}
}
The example above modifies the design of the p element. When its width is less than 700 pixels, the background-color is lightblue, otherwise it is lightgreen. Moreover, the font-size is adjusted. When a media query is true, the corresponding style sheet or style rules are applied, following the normal cascading rules.

The max-width feature is just of the several features that can be used in a media query. Here are some common features that you can also use:
Value Description
aspect-ratio Specifies the ratio between the width and the height of the display area
color Specifies the number of bits per color component for the output device
device-height Specifies the height of the device, such as a computer screen
device-width Specifies the width of the device, such as a computer screen
grid Specifies whether the device is a grid device or not
height Specifies the height of the display area, such as a browser window
max-height Specifies the maximum height of the display area, such as a browser window
max-resolution Specifies the maximum resolution of the device, using dpi or dpcm
max-width Specifies the maximum width of the display area, such as a browser window
min-height Specifies the minimum height of the display area, such as a browser window
min-resolution Specifies the minimum resolution of the device, using dpi or dpcm
min-width Specifies the minimum width of the display area, such as a browser window
orientation Specifies the whether the display is in landscape mode or portrait mode
resolution Specifies the resolution of the device, using dpi or dpcm
width Specifies the width of the display area, such as a browser window


Media queries can also be included in the link tag.

Example

<!--When the browser is between 0px and 640px wide, max-640px.css will be applied.-->
<link rel="stylesheet" media="(max-width: 640px)" href="max-640px.css">
<!--When the browser is 640px or wider, min-640px.css will be applied.-->
<link rel="stylesheet" media="(min-width: 640px)" href="min-640px.css">
<!--When the browser height is greater than the width, portrait.css will be applied.-->
<link rel="stylesheet" media="(orientation: portrait)" href="portrait.css">
<!--When the browser width is greater than the height, landscape.css will be applied.-->
<link rel="stylesheet" media="(orientation: landscape)" href="landscape.css">
